# Step 4: Add Data

To execute SQL statements that donâ€™t return rows (like `INSERT`, `UPDATE`, or `DELETE`), you use the `Exec` method.

## Instructions

### 1. Use Exec to Insert
Implement the `addAlbum` function:

```go
func addAlbum(alb Album) (int64, error) {
    result, err := db.Exec("INSERT INTO album (title, artist, price) VALUES (?, ?, ?)", 
        alb.Title, alb.Artist, alb.Price)
    if err != nil {
        return 0, fmt.Errorf("addAlbum: %v", err)
    }
    
    // Get the ID of the newly inserted row
    id, err := result.LastInsertId()
    if err != nil {
        return 0, fmt.Errorf("addAlbum: %v", err)
    }
    return id, nil
}
```

### 2. Call the function in main
```go
albID, err := addAlbum(Album{
    Title:  "The Modern Sound of Betty Carter",
    Artist: "Betty Carter",
    Price:  49.99,
})
if err != nil {
    log.Fatal(err)
}
fmt.Printf("ID of added album: %v\n", albID)
```

## Key Concepts
- **db.Exec**: Used for queries that perform an action but don't return data rows.
- **result.LastInsertId()**: Returns the integer generated by the database for the new row (common with `AUTO_INCREMENT` columns).
- **result.RowsAffected()**: Can be used to check how many rows were modified by an `UPDATE` or `DELETE`.
